# Dart Homework
## 프로젝트 개요
이 프로젝트는 Dart 언어를 활용한 간단한 프로그램 3종으로 구성되어 있습니다. 각 프로그램은 실제 생활에서 활용 가능한 기능들을 시뮬레이션하며, 예외 처리와 조건문, 컬렉션 사용법 등을 학습 목적으로 구현하였습니다.

## 포함된 프로그램
### 1. Grade Calculator (grade)

학생의 점수를 입력하면, 점수 범위에 따라 A, B, C 등급을 출력합니다.
- 예외 처리: 점수가 0~100 범위를 벗어나면 예외 발생

### 2. Shopping Cart Calculator (cart)

장바구니에 담긴 상품들의 가격을 합산하고, 만약 총 금액이 20000원 이상이면 10% 할인을 적용합니다.
출력: 장바구니 총 금액, 할인 금액, 최종 결제 금액
- 예외 처리:
  - 장바구니가 비어 있을 때
  - 장바구니에 존재하지 않는 상품이 포함될 때

### 3. Lotto Simulator (lotto / plusLotto)

1~45 사이 숫자 6개를 중복 없이 랜덤으로 뽑아 복권 번호를 발급합니다.
지정된 당첨 번호와 비교하여 1~3등 또는 당첨 실패 여부를 출력합니다.
발급 후 다음 로또 조회를 위해 번호를 초기화합니다.
(plusLotto 버전에서는 맞은 번호까지 함께 출력하도록 확장 기능 추가)


## 사용 방법
1. Grade Calculator
```dart
grade(84); // 예시: 84점
```

2. Shopping Cart Calculator
```dart
cart(["티셔츠", "바지", "모자", "티셔츠", "바지"]);
```

3. Lotto Simulator
``` dart
lotto();       // 기본 로또
plusLotto();   // 맞은 번호까지 출력되는 확장 로또
```

## 트러블슈팅 & 느낀점
이번 과제를 진행하면서 특히 장바구니 총 금액 계산 로직 구현과 예외 처리 시나리오 고민이 어려웠다.

### 장바구니 총 금액 계산
장바구니에 담긴 상품 리스트(List)와 가격표(Map)를 연결하여 합계를 구하는 방법을 처음에는 감이 잡히지 않았다. 반복문 선택과 Map에서 값을 안전하게 가져오는 방법을 고민했고, Dart 공식 문서를 참고해 안전하게 값을 가져오는 방식으로 해결했다. 이를 통해 Map과 List 활용 패턴과 컬렉션 자료형에 대한 이해를 높일 수 있었다.

### 예외 처리
프로그램에서 어떤 상황에서 예외가 발생할지 고민하며 장바구니가 비어 있을 때, 존재하지 않는 상품이 포함되었을 때, 점수가 0~100 범위를 벗어났을 때 등 다양한 경우를 고려했다. 단순히 코드를 구현하는 것이 아니라, 사용자가 잘못된 입력을 했을 때 프로그램이 안전하게 동작하도록 설계하면서 예외 처리의 중요성을 다시 한 번 느낄 수 있었다.

### 느낀점
이번 과제를 통해 컬렉션 활용, 반복문, 조건문, 예외 처리, 랜덤 데이터 처리 등 Dart에서 자주 쓰이는 기능을 실제 상황에 적용해 보는 연습을 충분히 할 수 있었다. 또한 문제를 분석하고 공식 문서를 참고하며 해결 방안을 찾는 과정에서 프로그래밍 문제 해결 능력도 한층 향상되었다고 느꼈다. 앞으로도 이렇게 하나씩 구현하면서 코드를 점점 더 안전하고 효율적으로 작성하는 연습을 계속해 나가고 싶다.
